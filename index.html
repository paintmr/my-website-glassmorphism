React To-Do List

  • This is a to-do list made with React.
  • Technologies used: HTML, CSS, JavaScript, React, React-Redux, Redux-Thunk, FontAwesome
  • A Store is created to store data.
  • The Provider from React-Redux connects the Store and the App.
  • Four UI components are made for user interaction: AddTodo, FilterSetter, TodoList (having a sub-component - Todo), and Alert. The user can add items through AddTodo. The user can choose which to-do list (all, the things to be done, the things finished) to show by clicking the buttons of the FilterSetter component. The user can change the status of the item(active, completed) by clicking the item(Todo) component of the TodoList component.
  • The AddTodo component can check if the user has entered anything and something that exists. When the user has entered something that exists, an alert layer pops up and then disappears itself after the user is reminded.
  • Four container components are made to connect Store and the four UI compoents. Each container component uses Connect from React-Redux to connect mapStateToProps and mapDispatchToProps with its corresponding UI component, so that each UI component can use Store data and dispaction actions easily.
  • Redux-Thunk is used to deal with asynchronous actions. When the TodoList component has mounted, an asynchronous action is dispatched to fetch data from a json file( called todos.json which is placed in the public/mock folder during development). Fetch is used to fetch the data. Then the data are stored in Store and shown by the TodoList component.
  • Sub-reducers are created based on Store State data: text, filter, todos, and alert.

    • Vue Node Express Upload Pictures

    • On this page, users can upload pictures to a Node server.
    • Technologies used: HTML, CSS, Bulma, Fontawesome, JavaScript, Vue, Axios, Node, Express, Multer, Sharp
    • This website includes two parts: the frond-end part and the back-end part. The front-end part has been developed with Vue, and the back-end has been developed with Express based on Node.
    • On the front-end side
    • The front-end part consists of three parts (three components): uploading a picture a time by clicking to choose a picture (the SimpleUpload component), uploading several pictures a time (the MultipleUploads component), and uploading a picture a time by dragging and dropping a picture to the uploading zone (the Dropzone component).
    • In the SimpleUpload and MultipleUploads components, there are both front-end and back-end verification processes to check if the file is empty, if the file is a picture, and if the file is too large.
    • In the MultipleUploads component, the user can see what files she has selected showing between the yellow choose-picture button and the blue send button. The user can click the cross button after each file to unselect the file. Invalid files (not pictures, too large) are in red and will not be sent to the server.
    • In the Dropzone component, there is a progress bar to show the uploading progress. When the picture has been uploaded, it can be seen underneath the Dropzone.
    • FormData() is used to process pictures to be uploaded.
    • Axios, async, await, try and catch are used to make requests to and handle responses from the server.
    • Bulma has many easy-to-use upload styles, and some of them are used in this project.
    • vue.config.js (including publicPath, assetsDir, and outputDir) is used to make sure the page works well after building, otherwise the index page would be blank.
    • On the server side
    • Nodemon is used for easier development.
    • The Cross-Origin Resource Sharing problem is solved to make information transfer smooth.
    • Multer is used for handling multipart/form-data. Multer is used on the server side to verify the uploaded file.
    • Sharp is used to process pictures uploaded. Sharp can resize and relocate pictures.
    • express.static() is used so that the front-end website can display pictures that have been uploaded.
